Structure du système
ADJOUA (Expéditrice)
1. Rédige le message
2. Calcule le hash SHA-256
3. Signe avec sa clé privée RSA
4. Chiffre le message avec AES-256
5. Chiffre la clé AES avec RSA (clé publique Koffi)
6. Transmet le paquet sécurisé
KOFFI (Destinataire)
1. Déchiffre la clé AES (RSA, clé privée)
2. Déchiffre le message (AES)
3. Vérifie la signature (clé publique Adjoua)
4. Compare les hash SHA-256
5. Valide et lit le message
Conclusion générale

Ce projet démontre concrètement comment sécuriser une communication numérique en combinant chiffrement, hachage et signature numérique.

La cryptographie classique (César, Vigenère) est utile à titre pédagogique pour comprendre les principes de base du chiffrement, mais elle est vulnérable aux attaques modernes.

Les systèmes actuels utilisent une combinaison d'algorithmes : AES pour la rapidité du chiffrement symétrique, RSA pour l'échange de clé et la signature, et SHA-256 pour l'intégrité des données.
